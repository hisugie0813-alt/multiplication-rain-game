# 구구단 산성비 게임 PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 제품명
**구구단 산성비 게임** (Multiplication Rain Game)

### 1.2 목적
초등학교 2학년 학생들이 곱셈구구를 재미있고 효과적으로 학습할 수 있도록 돕는 교육용 게임

### 1.3 타겟 사용자
- 주 사용자: 초등학교 2학년 학생 (8-9세)
- 부 사용자: 교사 및 학부모

### 1.4 핵심 가치
- **학습 효과성**: 반복 학습을 통한 구구단 암기 강화
- **재미 요소**: 게임화를 통한 학습 동기 부여
- **적절한 난이도**: 학생 수준에 맞는 단계별 학습

---

## 2. 기능 요구사항

### 2.1 게임 메커니즘

#### 2.1.1 기본 게임플레이
- **산성비 낙하**: 화면 상단에서 곱셈 문제가 적힌 물방울이 떨어짐
  - 예: "3 × 4", "2 × 7", "5 × 6"
  - 낙하 속도: 난이도에 따라 조절 가능
  - 동시 낙하 개수: 3-5개

- **정답 입력**: 화면 하단에 숫자 버튼(0-9) 배치
  - 학생이 답을 입력하고 확인 버튼 클릭
  - 키보드 입력 지원 (선택사항)

- **문제 제거**: 정답 입력 시 해당 물방울 소멸
  - 정답 효과음 및 시각 피드백
  - 점수 획득 (+10점)

- **게임 오버 조건**
  - 물방울이 화면 하단에 도달하면 생명 -1
  - 총 3개의 생명 제공
  - 모든 생명 소진 시 게임 종료

#### 2.1.2 난이도 선택
- **쉬움**: 2, 3, 4, 5단만 출제 / 낙하 속도 느림
- **보통**: 2-7단 출제 / 낙하 속도 중간
- **어려움**: 2-9단 출제 / 낙하 속도 빠름
- **단 선택 모드**: 특정 단만 집중 연습 (예: 7단만)

### 2.2 UI/UX 구성

#### 2.2.1 메인 화면
- 게임 타이틀 및 로고
- 게임 시작 버튼
- 난이도 선택 버튼
- 단 선택 모드 버튼
- 최고 점수 표시
- 설정 버튼 (음향 효과 ON/OFF)

#### 2.2.2 게임 화면
```
┌─────────────────────────────────────┐
│  점수: 120    생명: ♥♥♥    시간: 45초 │
├─────────────────────────────────────┤
│                                     │
│         🌧 3×4    🌧 7×2           │
│                                     │
│    🌧 5×6                           │
│                                     │
│              🌧 4×8                 │
│                                     │
│  ─────────────────────────────────  │
│  │  답 입력: [__]  │              │
│  │  [7][8][9]      │              │
│  │  [4][5][6]      │              │
│  │  [1][2][3]      │              │
│  │  [0]  [확인][X] │              │
└─────────────────────────────────────┘
```

#### 2.2.3 게임 종료 화면
- 최종 점수 표시
- 맞춘 문제 수 / 전체 문제 수
- 칭찬 메시지 (점수에 따라)
  - 100점 이상: "최고예요! 🌟"
  - 50-99점: "잘했어요! 😊"
  - 50점 미만: "다시 도전해보세요! 💪"
- 다시하기 버튼
- 메인으로 버튼

### 2.3 시각 디자인

#### 2.3.1 색상 팔레트
- **배경**: 하늘색 그라데이션 (#87CEEB → #E0F6FF)
- **물방울**: 투명한 파란색 (#4DA6FF)
- **버튼**: 밝고 친근한 색상 (주황색 #FFA500, 초록색 #4CAF50)
- **텍스트**: 크고 읽기 쉬운 검은색 폰트

#### 2.3.2 애니메이션
- 물방울 낙하 애니메이션 (부드러운 이동)
- 정답 시: 반짝이는 효과 + 물방울 사라짐
- 오답 시: 흔들림 효과
- 생명 감소 시: 하트 깜빡임

#### 2.3.3 폰트
- 크기: 최소 24px (가독성 확보)
- 스타일: 둥글고 친근한 고딕체
- 굵기: 중간-굵게 (Medium-Bold)

### 2.4 사운드

#### 2.4.1 효과음
- 정답 효과음: 경쾌한 "띠링~" 소리
- 오답 효과음: 부드러운 "띵동" 소리
- 물방울 낙하 효과음: 가벼운 빗소리
- 게임 오버: 안타까운 "아~" 소리

#### 2.4.2 배경음악
- 밝고 경쾌한 멜로디
- 반복 재생 (루프)
- 볼륨 조절 가능

## 3. 비기능 요구사항

### 3.1 성능
- 게임 로딩 시간: 3초 이내
- 입력 반응 시간: 즉각 반응 (50ms 이내)
- 프레임률: 최소 30fps 유지

### 3.2 호환성
- **플랫폼**: 웹 브라우저 (Chrome, Edge, Safari)
- **디바이스**: PC, 태블릿, 모바일
- **해상도**: 반응형 디자인 (최소 768px 이상)

### 3.3 접근성
- 색맹 사용자를 고려한 색상 대비
- 큰 터치 영역 (버튼 최소 44px × 44px)
- 간단하고 직관적인 UI

### 3.4 데이터 저장
- 최고 점수 로컬 저장 (LocalStorage)
- 난이도별 기록 저장
- 학습 진도 추적 (선택사항)

## 4. 개발 우선순위

### Phase 1 (MVP - 필수 기능)
1. ✅ 기본 게임 메커니즘 (물방울 낙하, 정답 입력)
2. ✅ 쉬움/보통 난이도 (2-5단)
3. ✅ 점수 및 생명 시스템
4. ✅ 기본 UI/UX
5. ✅ 효과음 및 시각 피드백

### Phase 2 (개선)
1. 🔲 어려움 난이도 및 단 선택 모드
2. 🔲 배경음악
3. 🔲 향상된 애니메이션
4. 🔲 최고 점수 저장

### Phase 3 (추가 기능)
1. 🔲 학습 진도 통계
2. 🔲 도전 과제 시스템
3. 🔲 친구와 점수 비교
4. 🔲 캐릭터 커스터마이징

## 5. 성공 지표 (KPI)

### 5.1 학습 효과
- 학생의 구구단 정답률 향상도
- 평균 게임 플레이 시간 (목표: 10분 이상)
- 반복 플레이율 (재방문율)

### 5.2 사용자 참여
- 주간 활성 사용자 수 (WAU)
- 게임 완료율
- 난이도별 선택 비율

### 5.3 만족도
- 학생 만족도 설문 (목표: 4.0/5.0 이상)
- 교사/학부모 추천 의향
- 평균 게임 세션 수

## 8. 리스크 및 대응 방안

### 8.1 리스크
1. **난이도 불균형**: 너무 쉽거나 어려울 경우 흥미 저하
2. **기술적 제약**: 다양한 디바이스에서 성능 이슈
3. **학습 효과 미달**: 게임만 하고 학습 효과 없음

### 8.2 대응 방안
1. **사용자 테스트 강화**: 실제 학생들과 반복 테스트
2. **성능 최적화**: 가벼운 라이브러리 사용, 코드 최적화
3. **학습 피드백 추가**: 틀린 문제 복습 기능 추가
